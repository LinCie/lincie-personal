## Challenges & Growth Areas

1. **TypeScript module scope gotcha** — `paper-tone.ts` and `local-time.ts` both declared `intervals` and `visibilityHandler` at module scope. TypeScript treated them as global-scope scripts (not ES modules) because neither had an `import` or `export`. Fix: `export {}` on both files. This is a gotcha specific to zero-import scripts in Astro's Vite pipeline.

2. **`data-reveal="corner"` discrepancy on BR folio — three stories documented, never fixed** — The spec says the folio should always be visible (no reveal animation), but the code has `data-reveal="corner"` which makes it start at opacity 0. Flagged in Story 5.2 review, carried through 5.3 and 5.4 without resolution. Pre-existing issue that has now been noted three times.

3. **`--dur-mark` token deferred for third consecutive epic** — Carried from Epic 3 → Epic 4 → Epic 5. Scope has grown from 2 files (Epic 3) → 3 files (Epic 4) → 4 files (Epic 5). Still low-impact but growing.

4. **Review patches clustered around robustness under edge conditions** — All 5 patches addressed "works fine in normal use, breaks under edge conditions" issues: double-interval on rapid visibilitychange, interval starts in hidden tab, clock drift from unaligned interval, stale folio text after navigation, single-viewport folio showing wrong TTT.

5. **Deferred items accumulating** — `deferred-work.md` now has entries from every epic across the entire project. Some items have been noted multiple times without resolution.

## Key Insights & Learnings

1. **Zero-import scripts need `export {}` for ES module scope** — Any `.ts` file in `src/scripts/` that has no `import` or `export` statements is treated as a global script by TypeScript. Adding `export {}` establishes module boundaries. This only surfaces when two such files coexist in the same bundle.

2. **The `intervals[]` + visibility-state pattern is the correct lifecycle for ambient polling** — Proven across two independent scripts. The pattern: start interval only if `!document.hidden`, clear stale intervals before restarting on `visible`, store `visibilityHandler` reference for cleanup removal.

3. **CSS transitions are sufficient for simple hover affordances** — Story 5.4 proved that `opacity` transitions with `250ms` timing don't need GSAP. The architecture threshold is correct: GSAP only for timelines, scroll-linking, sequencing, or physics.

4. **Pattern reuse compounds velocity** — Epic 5 delivered 4 stories in one day because every pattern was already proven: gate+cleanup lifecycle, component extraction, ScrollTrigger instance management, scoped CSS transitions. The investment in documenting patterns in Dev Notes pays dividends in later epics.

5. **Deferred items should have a promotion rule** — Items noted 3+ times across stories/epics should be promoted to action items. The `data-reveal="corner"` discrepancy and `--dur-mark` token both hit this threshold.
